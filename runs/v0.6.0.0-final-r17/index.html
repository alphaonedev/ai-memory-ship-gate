AI NHI analysis · Claude Opus 4.7

Writes now land (150 ok) but convergence metric is broken

Chaos source allowlist fix landed; writes succeed pre-kill. But `convergence_bound=0.03` revealed the harness's convergence math itself is wrong for kill-type faults.

What this campaign tested

Phase 4 with valid source, 50 cycles of kill_primary_mid_write.

What it proved (or disproved)

The product accepts `source="chaos"` writes; quorum succeeds for the 3 pre-kill writes in each cycle. But the published metric `total_ok / total_writes` is mathematically capped at ~2% for kill faults (primary dies at write 2 of 100), making the 0.995 threshold unreachable no matter how healthy the cluster is.

For three audiences

Non-technical end users

We can now write to the system in the chaos test. But the scoring system is rigged against us: it measures 'what fraction of attempts succeed' and the test deliberately kills the server after the second attempt. We need to measure 'of the attempts that succeeded, did the data actually survive?' That's the real signal.

C-level decision makers

A measurement-definition bug was masking a passing product. The test was asking the wrong question. Corrected metric is `of writes that 201'd, what fraction landed on the surviving peers`, which is the actual eventual-consistency contract.

Engineers & architects

run-chaos.sh line 221: `convergence_bound = ok/writes * 1000 | floor / 1000`. With `kill_primary_mid_write` firing at write 2 of 100, writes 3..99 hit a dead port — always fail. Ratio caps at ~2%. Right metric: `min(count_node1, count_node2) / total_ok` per cycle, aggregated. That answers 'did quorum survivors have the write?' which is the eventual-consistency guarantee.

Bugs surfaced and where they were fixed

  1. Phase 4 convergence metric measured uptime, not convergence

    Impact: 0.995 threshold unreachable for kill faults regardless of product correctness.

    Root cause: `ok/total_writes` ignores that most writes-during-kill are guaranteed to fail; doesn't examine whether survivors converged.

    Fixed in:

    • PR #312 (MERGED)

What changed going into the next campaign

r18 runs with the surviving-peer metric. Surfaces two deeper harness bugs (cumulative count pollution + partition_minority teardown race) — all closed in PR #312.
