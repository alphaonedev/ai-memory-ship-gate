First run after runner-driven SSH refactor

Baseline for the faster, cheaper methodology.

What this campaign tested

Same four phases, now orchestrated from a single GitHub Actions runner over SSH — droplets hold workload only.

What it proved (or disproved)

Runner-driven methodology completes a campaign in roughly 10 minutes at <$0.15 DO spend. The cost/time claims now published on the dashboard are measurable, not aspirational.

For three audiences

Non-technical end users

The tests got faster and cheaper — from roughly an hour to about fifteen minutes, and from about sixty cents a run to about ten cents. Nothing about what the tests check changed; only how they run.

C-level decision makers

Release-gate latency drops 6× and spend drops 6× from the candidate-era methodology. Velocity gain with no loss of signal.

Engineers & architects

commit f81bd76 moved orchestration onto the runner. Droplets still execute — SIGKILL / iptables still need local root — but the runner holds SCP/SSH so per-droplet boot costs vanish.

What changed going into the next campaign

Runs 11-14 iterate on the Phase 2 harness and uncover the federation post-quorum abort bug that was masking a correctness regression.
